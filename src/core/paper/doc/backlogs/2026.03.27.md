## Incidentes do Dia

### 1. Rate Limit GLM-5 (CRÍTICO)
- **O que aconteceu:** Cron job de monitoramento a cada 1 min consumiu cota da API do modelo
- **Impacto:** Sistema ficou 5h bloqueado (rate limit atingido)
- **Consequência:** Preço passou pelo alvo de venda (66,100) quando faltava apenas US$ 2.02 e não executou
- **Horário:** ~18:22 - ~23:00

### 2. Spam de Mensagens no Discord
- **O que aconteceu:** ~30 mensagens repetitivas de monitoramento
- **Impacto:** Poluição visual, consumo desnecessário de rate limit
- **Solução:** Usar `edit_message` para atualizar UM painel único

### 3. Feed Lag Yahoo vs Binance
- **O que aconteceu:** Diferença de preço de ~US$ 90-150 entre fontes
- **Exemplo:** Binance em 66,106 enquanto Yahoo mostrava 66,021
- **Impacto:** Perda de oportunidade de execução no alvo

### 4. Quantidades Fracionadas
- **O que aconteceu:** Usuário quis vender 0.1 BTC e recebeu erro
- **Why:** Sistema modelado apenas com `int` para quantidade

### 5. Saldo não Sincroniza
- **O que aconteceu:** Editar `paper_state.json` não reflete no servidor
- **Why:** Servidor carrega estado apenas na inicialização

## Backlog de Melhorias

### Prioridade ALTA

#### 1. Script Standalone de Monitoramento
- **Problema:** Monitoramento depende do modelo de IA (GLM-5)
- **Solução:** Criar script Python que roda independentemente
- **Benefício:** Zero consumo de cota da API do modelo
- **Arquivo sugerido:** `src/core/paper/scripts/monitor.py`

#### 2. Suporte a Quantidades Fracionadas
- **Problema:** Sistema só aceita inteiros
- **Solução:** Implementar `Decimal` ou `float` para quantidade
- **Impacto:** Permite operar 0.1 BTC, 0.5 ETH, etc.

#### 3. Endpoint de Depósito
- **Problema:** Editar JSON não reflete no servidor
- **Solução:** `POST /deposito {valor}`
- **Benefício:** Adicionar fundos sem reiniciar servidor

### Prioridade MÉDIA

#### 4. Multi-Feed de Preços
- **Problema:** Fonte única (Yahoo Finance) com lag
- **Solução:** Agregar Binance, Coinbase, Yahoo
- **Benefício:** Preços mais precisos, detector de arbitragem

#### 5. Sistema de Alertas Condicionais
- **Comando:** `/alerta criar ticker operador valor acao`
- **Exemplo:** `/alerta criar BTC-USD >= 67000 venda 1`
- **Benefício:** Automação sem intervenção manual

#### 6. Range Trading
- **Problema:** Sistema só executa em preço exato
- **Solução:** `/range criar ticker min max qtd_total tranches`
- **Benefício:** Executar em range de preços

### Prioridade BAIXA

#### 7. Sistema de OHLC/Candles
- **Problema:** Não há histórico de Open-High-Low-Close
- **Solução:** Coletor de candles (1m, 5m, 15m, 1h, 1d)
- **Benefício:** Identificar suportes/resistências

#### 8. Análise de Canal/Tendência
- **Problema:** Não rastreia máximas/mínimas
- **Solução:** Rastreamento de HH/HL/LH/LL
- **Benefício:** Detectar tendência (alta/baixa/lateral)

#### 9. Detector de Arbitragem
- **Problema:** Diferença de preço entre corretoras
- **Solução:** Monitor de spread com alertas
- **Benefício:** Lucro livre de risco

#### 10. Intervalo de Polling Dinâmico
- **Problema:** Intervalo fixo (1 min)
- **Solução:** Adaptar conforme distância do alvo
  - Longe (>1%): 5 min
  - Perto (<1%): 2 min
  - Muito perto (<0.1%): 30s

## Lições Aprendidas

1. **Não usar modelo de IA para monitoramento contínuo**
   - Rate limit é caro e bloqueia operações
   - Usar scripts standalone ou server-side

2. **Usar `edit_message` para painéis dinâmicos**
   - Uma mensagem que atualiza > 30 mensagens novas
   - Economiza rate limit do Discord

3. **Multi-feed é essencial para trading**
   - Lag de US$ 90-150 pode perder oportunidades
   - Agregar múltiplas fontes aumenta precisão

4. **Conversão cambial não implementada**
   - Sistema trata US$ como R$ (1:1)
   - Precisa implementar taxa de câmbio real

5. **Quantidades fracionadas são obrigatórias para cripto**
   - BTC, ETH, etc. operam com casas decimais
   - `int` não é suficiente
