# TASK-2025-12-22 — Rodar Discovery Skybridge (Snapshot + Score)

## Objetivo

Executar a descoberta das entidades Skybridge em `B:\_repositorios` e gerar o relatório comparativo.

## Referências

Utilize as documentações existentes para tirar dúvidas, como o guia: `B:\_repositorios\skybridge\docs\playbook\PB000 - Discovery da Skybridge via Snapshot + Scoring.md`.

## Instruções

Use a ferramenta snapshot existente (B:\_repositorios\pyro\README_SNAPSHOT.md) como referência de estrutura e formato.

Varra o diretório B:\_repositorios com profundidade máxima de 5 níveis, tratando cada diretório candidato como uma entidade potencial da Skybridge (variações de nome: skybridge, sky-bridge, sky_bridge, sky, bridge).

Objetivo: identificar implementações, tentativas ou fragmentos da Skybridge espalhados pelos repositórios.

### Metodologia

- A varredura inicial deve usar apenas nomes de pastas, nomes de arquivos e extensões (evitar leitura completa de arquivos nesta fase).
- Ler conteúdo apenas de README*, pyproject.toml ou arquivos de configuração quando necessário como evidência.

### Cálculo de score

- Atribuir pesos por tipo de arquivo encontrado na entidade:
  - .py (código)
  - .md (documentação)
  - .toml / .txt (configuração)
  - .yml / .yaml / .json (data, infra, OpenAPI, workflows)
- Atribuir peso adicional para marcadores fortes:
  - pastas chamadas skybridge, modules/skybridge, src/skybridge
  - README mencionando Skybridge
  - presença de CLI (__main__.py, cli.py)
- Penalizar entidades muito genéricas (monolitos) sem marcadores fortes.

### Resultado esperado (relatório)

1) Lista ordenada de entidades candidatas (por score).
2) Evidências encontradas por entidade (arquivos, pastas, marcadores).
3) Recomendações:
   - provável núcleo da Skybridge
   - implementações paralelas ou abandonadas
   - fragmentos reaproveitáveis (funcionalidades que cada entidade traz)
4) Tabela comparativa de recursos por entidade:
   - código
   - documentação
   - CLI
   - snapshot/diff
   - dominios (File, Rota, Mensagem, etc.)
   - integrações (git, openapi, etc.)

### Finalidade

Gerar um diagnóstico claro para unificar a Skybridge e retomar o desenvolvimento de forma intencional.
